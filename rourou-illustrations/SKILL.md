# ROUROU 纸本正文配图

## 核心定位

为中文文章设计和生成 16:9 横版正文配图。把文章中的关键判断、流程、结构、状态或隐喻，变成一张有纸张触感、略微怪诞、简洁清楚的手绘解释图。

默认视觉 IP 是 ROUROU（揉揉）：一个带蓝色回形针、红色缝线、直腿和弹簧腿的暖白揉皱纸张小工匠。让 ROUROU 承担核心动作，不要把它当吉祥物或角落装饰。

## 按任务读取参考

- 生成或修改任何含角色的图片：必须读取 `references/rourou-ip.md` 和 `references/style-dna.md`。
- 生成单张图片：同时读取 `references/prompt-template.md`。
- 实际生图或改图：同时读取 `references/generation-backends.md`，按 Codex、WorkBuddy/Claude Code 或纯提示词模式选择后端。
- 规划隐喻或构图：读取 `references/composition-patterns.md`。
- 出图后：读取 `references/qa-checklist.md` 并检查。
- `assets/ip-reference/01-turnaround.png` 是每次生图必用的严格角色参考图；再按任务从同目录选择最多一张表情、动作或变形参考图。
- `assets/examples/` 只作低频视觉校准，不进入默认生成路径；不要复刻其中的构图、物件或标注。

## 工作流

### 1. 消化正文

提炼核心观点、认知转折、适合视觉化的段落和不需要配图的段落。不要平均配图；优先选择核心判断、输入输出闭环、分流、前后对比、承接路径、常见坑或角色状态变化。

### 2. 先出配图策略

如果用户只要求分析配图，先给 shot list。每张图写清：放置段落、主题、核心意思、结构类型、ROUROU 的核心动作、建议纸本道具和短中文标注。默认 4-8 张；短文 1-3 张，长文通常不超过 9 张。

### 3. 单张生成

如果用户明确要求生成图片，先按 `references/generation-backends.md` 检查当前宿主可用的图像生成或编辑能力。每张图单独生成，不要把多张图拼成一张。

- Codex 有 `image_gen` 时，直接调用并传入严格角色参考图。
- WorkBuddy/Claude Code 中，先发现当前会话已启用的图像工具或 MCP 能力，再按其参数传入参考图和完整提示词；不要假设 Codex 工具名存在。
- 当前宿主没有可用图像后端时，交付逐张可执行的提示词包、参考图绝对路径和建议文件名，并明确说明尚未实际生成图片；不要伪造产出。

每张图必须：

- 使用 `assets/ip-reference/01-turnaround.png` 锁定 ROUROU 身份。
- 使用 16:9 横版、纯白背景、暖白纸本角色和手绘炭黑线。
- 保留 ROUROU 的雾霾蓝回形针、陶土红斜向缝线、直腿与弹簧腿。
- 只选 1-2 种辅助莫兰迪色，彩色面积约 10%-20%。
- 保留大量留白，让 ROUROU 推动核心结构运转。
- 禁止 PPT、商业矢量吉祥物、儿童卡通、复杂架构、高饱和配色、类型标题、Logo 和水印。

每次从当前文章重新发明一个物理隐喻，不要复刻参考图或历史案例的构图。

### 4. 检查与迭代

按 `references/qa-checklist.md` 检查角色身份、参与度、构图、纸张材质、莫兰迪色和文字。角色走样时优先带 `01-turnaround.png` 重生成；局部文字或多余标识才使用局部编辑。

### 5. 保存交付

在 workspace 内工作时，把最终图保存到 `assets/<article-slug>-illustrations/`，按顺序命名为 `01-topic-name.png`、`02-topic-name.png`。不覆盖已有资产，除非用户明确要求替换。

## 输出口径

生成前策略要短而准。生成后说明数量、用途、保存路径和可选图，不长篇解释风格理论。
