# 生成后端适配

视觉规范与宿主工具解耦。先确定当前宿主和可用能力，再执行生图；不要因为工具不同而删减角色身份、画风或 QA 约束。

## Codex

有内置 `image_gen` 时：

1. 读取 `prompt-template.md` 并填满所有变量。
2. 把 `assets/ip-reference/01-turnaround.png` 作为必选参考图。
3. 按任务最多再加入一张表情、动作或变形参考图。
4. 新图使用生成模式；修改已有图使用编辑模式，并同时传入待编辑图与角色参考图。
5. 每次只请求一张独立图片，生成后执行 QA。

## WorkBuddy / Claude Code

WorkBuddy 运行在 Claude Code 环境时，不要调用或声称调用 Codex 专属的 `image_gen`。

1. 先检查当前会话暴露的工具和 MCP 服务，寻找支持“图像生成”“图像编辑”“参考图输入”的能力。
2. 如果 WorkBuddy MCP Gateway 可用，先用其能力搜索工具查询 image generation、image editing、reference image；读取返回的真实参数结构后再执行，不要猜工具名或字段。
3. 优先选择能同时接收文本提示词与本地参考图片的后端。把 `01-turnaround.png` 作为严格身份参考，并按任务最多再附一张参考图。
4. 若后端只接受文本而不能读取本地图片，继续生成前必须告诉用户角色一致性会降低；同时把 `prompt-template.md` 中完整的身份不变量写入提示词。
5. 若后端不能直接保存到工作区，报告其实际返回位置或资源标识，不虚构本地文件路径。
6. 完成后按 `qa-checklist.md` 检查；若当前工具无法查看结果，明确标记“待视觉 QA”。

`agents/openai.yaml` 只是 Codex 的界面元数据。WorkBuddy/Claude Code 可以忽略该文件，实际行为以 `SKILL.md` 和 `references/` 为准。

## 无图像后端时的提示词模式

当当前宿主没有图像生成或编辑能力时，不中断配图策划。为每张图交付：

- 序号、主题和放置段落
- 由 `prompt-template.md` 填充完成的独立提示词
- 必选角色参考图的绝对路径
- 可选第二参考图的绝对路径
- 建议输出比例 `16:9`
- 建议文件名，如 `01-minimum-loop.png`
- 状态：`PROMPT_ONLY_NOT_GENERATED`

不要使用“已生成”“已保存”或“见附件”等措辞。

## 后端能力最低要求

优先级从高到低：

1. 文本 + 多张参考图 + 图像编辑 + 可查看输出
2. 文本 + 单张参考图 + 可查看输出
3. 仅文本生成
4. 只输出提示词，不实际生图

后端变化只影响执行方式，不改变 ROUROU 的固定身份、莫兰迪色比例、16:9 构图和非 PPT 约束。
